## 물리 
osi 모델에서 가장 하위에 위치하는 계층 
실제 장치를 연결 하는데 필요한 전기적 , 물리적 세부 사항 정의 
즉 실제로 만질 수 있는것 
### 프로토콜 종류 
x.21 
RS-232 
### 주요장비 
-네트워크 인터페이스 카드 (NIC)
    네트워크와 컴퓨터 사이의 물리적 연결 
    네트워크에서 데이터 전송 그리고 수신 하는 역할 
-리피터
    신호를 증폭하여 전송 거리를 연장하거나 신호의 품질을 유지 하는데 사용 
- **허브** 
    여러 컴퓨터 또는 네트워크 장치를 연결하여 네트워크를 형성하는데 사용 
    데이터를 받으면 그 데이터를 모든 연결된 장치로 브로드캐스트 하여 전송

- 광섬유

## 연선종류 
- UTP 
    쉴드로 보호되지 않은 연선이다 가장 일반적으로 사용되는 형탸 
    이더넷 , 네트워크 전화 시스템에 주로 사용 된다 
    설치 간단 비용 저렴 
- FTP 
    전체 케이블을 둘러싸는 얇은 알루미늄 포일 쉴드로 보호되는 형태 
- STP 
    각쌍과 전체 케이블이 알루미늄 쉴드로 보호되는 형태 
### 연선 배선 방식 표준 
## 다이렉트 
- 스트레이트 스루 케이블 
    구리 선 여덟개를 동일한 순서로 연결

- 크로스 오버 케이블 
    서로 같은 장치를 연결 할때 사용 된다 
    568A-568B

## 데이터링크 
    두번째 계층 
    데이터의 프레임화 물리주소(MAC 주솔)를 이용한 통신 오류 검출 등의 기능을 수행한다 

### 통신방법 
- CSMA/CD 
    이더넷과 같은 배포된 물리 네트워크에서 주로 사용된다 
    충돌 감지를 통해 여러 장치가 동시에 데이터를 전송하지 않도록 제어한다 
    충돌을 감지하고 대응하기 위해 물리적으로 데이터 전송을 감지하고 정지 시키는 기술을 사용한다 
- CSMA/CA
    무선 네트워크에서 사용되는 프로토콜 주로 WI-Fi와 같은 무선 네트워크 기술에서 사용된다 
    물리적인 충돌을 감지하는것이 어려운 무선 환경에서 충돌을 피하기 위한 방법을 제공 
    전송하기 전에 채널이 사용 가능한지 확인하고 충돌을 피하기 위한 절차를 따른다 
### 주요장비 
- 브리지 
    두개의 LAN을 상호 접속해 주는 통신망 연결 장치로서 데이터 링크 계층에서 동작 
    브리지는 통신망의 범위길이 확장 
    서로 다른 통신망 구조의 통신망을 연결 할때 등에 사용 가능 
    2계층 부터 주소 분류 가능 
- 스위치 
    허브와 달리 데이터 충돌이 발생하지 않는다 
    이는 스위차가 각 포트를 독립적인 충돌 도메인으로 관리하며, 한 번에 여려 통신을 동시에 처리할 수 있기 때문이다 

### MAC 주소 
    네트워크 장비가 네트워크 상에서 고유하게 식별되기 위해 사용되는 고유한 식별자 
    MAC 주소는 네트워크 인터페이스 카드에 할당되어 있으며, 이 주소를 통해 네트워크에서 데이터를 주고 받는 장비를 식별 
### MAC 주소 형식 
    48비트 
    16진수로 표현 
    주소는 두 부분으로 나뉜다 첫번째는 제조사 식별 두번째는 고유한 장비

### 스위치 기능 
-MAC 주소 테이블 
    스위치는 연결된 장치들의 MAC 주소를 MAC 주소 테이블에 저장 한다 
    테이블은 스위차가 프레임을 더 빠르고 효율적으로 전달할 수 있게 돕는다 
- 플러딩 
    스위치는 처음 부팅할때 네트워크 관련 정보가 없다 
    이런 상태일 경우 허브처럼 모든 포트로 패킷을 전송한다 
    스위치는 LAN에서 동작하므로 자신이 정보가 없더라도 누가 가지고 있다고 생각하고 작업 수행 
- 어드레스 러닝 
    패킷이 특정 포트에 들어오면 스위치는 해당 패킷의 출발지 MAC
